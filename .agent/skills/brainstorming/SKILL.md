# Brainstorming Skill

Use this skill when transforming ideas into designs and specs through collaborative dialogue.

## When to use this skill

-   Use when transforming ideas into designs and specs through collaborative dialogue.

## Overview

Turn ideas into fully formed designs through natural dialogue. Check project state first, ask questions one at a time, present 2-3 approaches with trade-offs.

## Key Principles

-   One question at a time
-   Multiple choice preferred
-   YAGNI ruthlessly
-   Explore 2-3 alternatives
-   Present design in 200-300 word sections
-   Validate each section before continuing

## After Design

-   Write to `docs/plans/YYYY-MM-DD-<topic>-design.md`
-   Use git worktrees for isolated implementation
-   Use writing-plans skill for detailed implementation plan
